# Abgabe-Post — OptiTax

Direkt kopierbarer Post für die Kategorie „Building Challenge" in der
SKAILE Academy. Vor dem Absenden nur noch den Demo-Link ersetzen.

```
ABGABE — OptiTax

Problem: Solo-Selbstständige bekommen Rechnungen, Zahlungsbelege und
Abo-Nachweise getrennt per E-Mail — das kostet Zeit, Originale fehlen und
Kosten werden doppelt oder gar nicht erfasst.

Demo: DEMO_LINK_EINFÜGEN

Repo: https://github.com/SkyiDExX/building-challenge-2

Neu in der Challenge entstanden:
- Plansteuerbarer, zustandsabhängiger Agent mit protokollierten
  Planrevisionen und getesteten Plan-Invarianten
- Sichere PDF- und EML-Verarbeitung mit Kostenvorgängen, Originalansichten
  und Duplikatschutz
- Produkt- und Abo-Profil, append-only Review-Korrekturen mit Neubewertung
  und prüfbarer CSV-Export (genau eine Kostenzeile pro Vorgang)

Stack: Claude Code · Python + SQLite + pypdf · Standardbibliothek für
E-Mail und lokalen Webserver · HTML/CSS/JS · komplett lokal, keine
externen Dienste
```

## Optionaler Community-Text (max. 60 Wörter)

Eine Rechnungs-E-Mail rein — OptiTax erkennt Produkt, Rechnung und
Zahlungsnachweis, verbindet sie zu einem Kostenvorgang und verhindert
Doppelzählungen. Ein regelbasierter Agent plant pro Eingang seine
Prüfwerkzeuge, revidiert bei neuer Evidenz und sagt ehrlich, was fehlt:
„Rechnung anfordern" statt raten. Komplett lokal, ohne API-Keys, mit
unveränderten Originalen und nachvollziehbarem Audit.
